## 2. 核心页面清单

| 页面 | 路由 | 功能描述 |
|------|------|----------|
| 首页 | `/` | 展示平台介绍、三条路线概览、最新用户 |
| 注册 | `/auth?mode=register` | 用户注册表单 |
| 登录 | `/auth?mode=login` | 用户登录表单 |
| 新建旅程 | `/journey/new` | 选择路线、填写基本信息 |
| 照片上传 | `/journey/:id/photos` | 上传GPS照片、查看验证状态 |
| 路线地图 | `/journey/:id/map` | Leaflet地图展示轨迹点 |
| 证书成就 | `/journey/:id/certificate` | 生成电子证书 |
| 个人中心 | `/profile` | 用户信息、旅程列表 |

## 3. 主要交互流程

### 流程A: 新用户首次使用
```
首页 → 点击[注册] → 填写注册表单 → 注册成功 → 自动登录 → 
提示"创建第一个旅程" → 新建旅程 → 上传照片 → 查看地图 → 申请证书
```

### 流程B: GPS照片验证流程
```
上传照片 → 系统提取EXIF GPS → 与路线GeoJSON比对 →
计算距离阈值(≤500m) → 通过/不通过 → 更新照片状态
```

### 流程C: 证书申请流程
```
旅程验证点≥10 → 点击[申请证书] → 生成证书页 → 可保存/分享
```

## 4. 关键技术点提示

### 移动端适配
- 移动优先设计，所有组件支持触摸操作
- 照片上传支持摄像头直接拍摄
- 地图支持手势操作（双指缩放、拖拽）
- 表单输入适配移动端键盘

### 地图组件选型
- **Leaflet**: 轻量级，支持移动端，插件丰富
- **OpenStreetMap**: 免费地图瓦片
- **Mapbox**: 可选，有更好的样式但需API Key

### GPS验证算法
```javascript
// Haversine公式计算两点间距离
function haversineDistance(lat1, lon1, lat2, lon2) {
  const R = 6371000; // 地球半径(m)
  const dLat = toRad(lat2 - lat1);
  const dLon = toRad(lon2 - lon1);
  const a = Math.sin(dLat/2) * Math.sin(dLat/2) +
            Math.cos(toRad(lat1)) * Math.cos(toRad(lat2)) *
            Math.sin(dLon/2) * Math.sin(dLon/2);
  const c = 2 * Math.atan2(Math.sqrt(a), Math.sqrt(1-a));
  return R * c;
}

// 验证逻辑：距离 ≤ 500m 视为通过
```

### EXIF提取
- 使用 exif-js 或 piexifjs 库
- 关键字段：GPSLatitude, GPSLongitude, DateTimeOriginal

### 下拉刷新
- 移动端列表页支持下拉刷新最新数据
- Web端可使用 Pull-to-refresh  polyfill

### 文件上传
- 移动端优先：支持 camera capture
- 限制：单文件≤10MB，支持jpg/png
- 预览：上传前显示缩略图

## 5. 关键设计决策

| 决策项 | 选择 | 理由 |
|--------|------|------|
| 移动端方案 | PWA / 响应式 | 减少开发成本 |
| 地图瓦片 | OSM | 免费、无需API Key |
| 照片存储 | 客户端Base64 / 未来OSS | MVP阶段简化 |
| 离线支持 | 可选V2 | 边境地区可能无信号 |
| 证书格式 | HTML5 Canvas生成图片 | 跨平台兼容 |
