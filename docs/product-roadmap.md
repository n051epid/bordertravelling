# 产品规划

**产品**：环中国边境线旅行记录平台（BorderTravelling）
**版本**：v1.0
**更新日期**：2026-04-18

---

## 产品定位

小众爱好者的自娱自乐平台，记录真实环中国边境线旅程。

**核心价值**：
- 真实记录 — GPS照片 + 时间戳 = 防伪验证
- 荣誉认证 — 证书体系，赋予完成仪式感
- 社区连接 — 同类人群的分享与互动

---

## 用户故事

### 作为游客（未注册）
- 我可以在地图上查看 G219/G331/G228 三条路线信息
- 我可以浏览其他用户的公开旅程记录和照片
- 我可以了解"环边境"是什么

### 作为注册用户
- 我可以上传旅途照片（自动提取GPS）
- 我可以为照片添加地点、备注
- 我可以记录自己的旅程进度（进行中/已完成）
- 我可以给自己的照片评论/回复
- 我可以获得阶段性成就徽章
- 我可以在完成全部路线后申请证书

## 版本规划

### v0.1 — MVP（4-6周）

**目标**：最小可运行版本，验证核心假设

| 模块 | 功能 | 优先级 |
|------|------|--------|
| 用户系统 | 手机号登录 | P0 |
| 照片模块 | 上传 + GPS提取 + 地理位置标注 | P0 |
| 地图模块 | 三条路线展示 + 点击弹窗 | P0 |
| 旅程记录 | 个人旅程时间线 | P0 |
| 个人主页 | 展示已记录路段 | P0 |

**技术依赖**：
- 前端：Next.js（SSR + 响应式）
- 后端：Node.js + Express + TypeScript
- 数据库：SQLite（快速启动）/ PostgreSQL（后续迁移）
- 文件存储：本地 /data/apps/（或阿里云OSS）
- 地图：Leaflet + OpenStreetMap

**MVP不包含**：
- 证书系统
- 评论/回复
- 成就徽章
- 微信/第三方登录
- 小程序

## 风险与应对

| 风险 | 影响 | 应对 |
|------|------|------|
| GPS照片隐私顾虑 | 用户不愿上传 | 支持关闭GPS、模糊位置 |
| 证书权威性不足 | 用户不认可 | 强调社区认证、联合KOL |
| 小程序非必要 | 资源浪费 | MVP不考虑，先做Web验证 |
| 用户增长慢 | 社区冷清 | KOL合作、游记投稿 |
